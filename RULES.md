# Правила разработки проекта CopyMe

## Тестирование и запуск локального сервера
При разработке игры используется локальный многопоточный веб-сервер (`python3 server.py`) и Cloudflare Tunnel для демонстрации проекта. Использование многопоточного сервера (вместо стандартного `python3 -m http.server`) решает проблему зависаний при множественных параллельных запросах браузера.

В связи с этим введено **ОБЯЗАТЕЛЬНОЕ ПРАВИЛО**:
1. **Проверка работоспособности**: После каждого значимого изменения кода, добавления новых функций или обновления ссылки необходимо проактивно проверять жив ли локальный сервер (например, с помощью `curl -I http://localhost:8888`).
2. **Перезапуск**: Если сервер перестал отвечать, его необходимо немедленно принудительно завершить (`pkill -f "python3 server.py"`) и запустить заново (`python3 server.py &`), чтобы ссылка для тестирования всегда оставалась активной.
3. **Обновление ссылки**: При перезапуске туннеля Cloudflare всегда убеждаться, что локальный сервер тоже работает корректно.
